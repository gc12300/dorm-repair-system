### 对话 3：注册事务 — 多表联动

**用户**：注册时要同时创建 `User` 和 `Student`（或 `Admin`），保证事务一致性。

**助手**：加 `@Transactional`，先插 User，再按角色插 Student 或 Admin。学生必须选宿舍楼。

```java
@Transactional
public Result<Object> register(RegisterDTO dto) {
    if (userMapper.selectCount(
        new LambdaQueryWrapper<User>().eq(User::getUsername, dto.getUsername())) > 0) {
        return Result.error(400, "用户名已存在");
    }
    User user = new User();
    user.setUsername(dto.getUsername());
    user.setPassword(dto.getPassword());
    user.setRole(dto.getRole());
    userMapper.insert(user);

    if ("ADMIN".equals(dto.getRole())) {
        Admin admin = new Admin();
        admin.setUserId(user.getId());
        adminMapper.insert(admin);
    } else {
        if (dto.getDormitoryId() == null)
            return Result.error(400, "学生必须选择宿舍楼");
        Student student = new Student();
        student.setUserId(user.getId());
        student.setDormitoryId(dto.getDormitoryId());
        studentMapper.insert(student);
    }
    return Result.success("注册成功");
}
```

**用户**：如果中途报错，前面的插入会自动回滚吗？

**助手**：对，`@Transactional` 默认 RuntimeException 回滚，MyBatis Plus 的 insert 抛异常就会触发。

## 二、调试后端错误

### 对话 1：UPDATE 中文乱码 — "待处理" 变成 "???"

**用户**：管理员处理报修后，数据库里的 status 字段变成了 `???`，怎么回事？

**助手**：看一下数据库字符集配置。

**用户**：INSERT 中文是正常的，只有 UPDATE 乱码。难道不是同一个连接池？

**助手**：排查了一下——MySQL CLI INSERT 正常，JDBC INSERT 正常，MySQL CLI UPDATE 正常，JDBC UPDATE 全部乱码。hex 值全是 `3F3F3F`（即 `???`）。

**结论**：MySQL 5.5 服务器 `character_set_server=latin1`，Connector/J 8.x 的 PreparedStatement 参数会走服务端字符集编码，UTF-8 多字节字符被降级成 `?`。INSERT 正常是因为参数走客户端编码，而 UPDATE 走了服务端编码路径。换 MySQL 8.0 + utf8mb4 彻底解决。

### 对话 2：端口冲突 — 8080 / 3306 被占用

**用户**：Docker 容器启动报错 `bind: address already in use`，端口 3306 和 8080 都被占了。

### 对话 4：部署报错 "No static resource ."

**用户**：部署后访问报 500 错误：`No static resource .`，页面打不开。

**助手**：这是因为 Vue Router 用了 history 模式，Spring Boot 找不到对应静态文件就 404 了。需要配置 SPA 回退，把非 API 请求都返回 index.html。

```java
@Configuration
public class WebConfig implements WebMvcConfigurer {
    @Override
    public void addResourceHandlers(ResourceHandlerRegistry registry) {
        registry.addResourceHandler("/**")
                .addResourceLocations("classpath:/static/")
                .resourceChain(true)
                .addResolver(new PathResourceResolver() {
                    @Override
                    protected Resource getResource(String resourcePath,
                            Resource location) throws IOException {
                        Resource resource = location.createRelative(resourcePath);
                        if (resource.exists() && resource.isReadable())
                            return resource;
                        return location.createRelative("index.html");
                    }
                });
    }
}
```

**用户**：访问正常了。

### 对话 3：docker-compose 编排 App + MySQL

**用户**：MySQL 和 App 用 docker-compose 一起启动，怎么写？

**助手**：关键点——MySQL 不对外暴露端口，用 `depends_on` + `condition: service_healthy` 确保启动顺序，环境变量覆盖数据源 URL。

```yaml

  mysql:
    image: mysql:8.0
    container_name: dorm-mysql
    environment:
      MYSQL_ROOT_PASSWORD: 123456
      MYSQL_DATABASE: dorm_repair
    volumes:
      - mysql_data:/var/lib/mysql
      - ./backend/sql/schema.sql:/docker-entrypoint-initdb.d/01-schema.sql
    command: --character-set-server=utf8mb4 --collation-server=utf8mb4_unicode_ci
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost"]
      interval: 10s
      timeout: 5s
      retries: 5

  app:
    build: ./backend
    container_name: dorm-app
    ports:
      - "8080:8080"
    environment:
      SPRING_DATASOURCE_URL: jdbc:mysql://mysql:3306/dorm_repair?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai&useSSL=false&allowPublicKeyRetrieval=true
      SPRING_DATASOURCE_USERNAME: root
      SPRING_DATASOURCE_PASSWORD: 123456
    depends_on:
      mysql:
        condition: service_healthy

volumes:
  mysql_data:
```

**用户**：`SPRING_DATASOURCE_URL` 里 host 写 `mysql` 而不是 `localhost`，是因为 Docker DNS？

**助手**：对，Docker Compose 会自动创建网络，服务名就是 hostname，容器间通过服务名互相访问。

## 四、集成大模型 API（AI 模块）

### 对话 1：DeepSeek API 通用调用封装

**用户**：帮我封装一个通用的 DeepSeek API 调用方法。

**助手**：核心是 `callDeepSeek` 方法，传 system prompt 和 user message，带重试机制（最多重试 2 次，间隔 1 秒）。

```java
@Service
public class AiService {
    private static final int MAX_RETRIES = 2;

    private String callDeepSeek(String systemPrompt, String userMessage) {
        String url = ds.getBaseUrl() + "/v1/chat/completions";
        Map<String, Object> body = Map.of(
            "model", ds.getModel(),
            "temperature", 0.3,
            "max_tokens", 1024,
            "messages", List.of(
                Map.of("role", "system", "content", systemPrompt),
                Map.of("role", "user", "content", userMessage)
            )
        );

        for (int attempt = 0; attempt <= MAX_RETRIES; attempt++) {
            try {
                HttpHeaders headers = new HttpHeaders();
                headers.set("Authorization", "Bearer " + ds.getApiKey());
                HttpEntity<Map<String, Object>> entity = new HttpEntity<>(body, headers);
                Map<String, Object> response = restTemplate.postForObject(
                    url, entity, Map.class);

                List<Map<String, Object>> choices =
                    (List<Map<String, Object>>) response.get("choices");
                Map<String, Object> message =
                    (Map<String, Object>) choices.get(0).get("message");
                String content = (String) message.get("content");

                // 去掉 ```markdown 包裹
                if (content.startsWith("```"))
                    content = content.replaceAll("```\\w*\\n?", "")
                                     .replaceAll("\\n```", "");
                return content.trim();
            } catch (RestClientException e) {
                if (attempt == MAX_RETRIES) return null;
                Thread.sleep(1000);
            }
        }
        return null;
    }
}
```

**用户**：为什么要去掉 markdown 代码块包裹？

**助手**：大模型习惯把 JSON 或 SQL 包在 ` ```json ... ``` ` 里返回，代码解析时需要剥离。

### 对话 2：管理员 NL2SQL — 自然语言查数据

**用户**：管理员想用自然语言查询数据库，比如"最近一周有多少待处理的报修"。

**助手**：用 AI 把自然语言转成 SQL，但要加安全限制——system prompt 约束只生成 SELECT，代码层再做 `startsWith("SELECT")` 校验，失败降级到预定义查询。

```java
public Nl2SqlResult nl2SqlQuery(String query) {
    String systemPrompt = """
        你是MySQL数据库助手。数据库 dorm_repair 包含以下表：
        repair (id, student_id, title, description, status,
                admin_id, handle_desc, create_time)
          - status取值: 待处理, 处理中, 已完成, 已取消
        student (id, name, student_no, phone, dormitory_id)
        admin (id, name, phone)
        dormitory (id, name)
        rating (id, repair_id, rating, review)

        将用户自然语言转为单条SELECT。要求：
        1. 只生成SELECT，禁止INSERT/UPDATE/DELETE
        2. 返回JSON: {"sql":"...", "explanation":"..."}
        """;

    String response = callDeepSeek(systemPrompt, query);
    if (response == null) return fallbackNl2Sql(query);

    Map<String, Object> map = objectMapper.readValue(response, Map.class);
    String sql = (String) map.get("sql");

    if (sql == null || !sql.trim().toUpperCase().startsWith("SELECT"))
        return fallbackNl2Sql(query);

    List<Map<String, Object>> data = jdbcTemplate.queryForList(sql);
    return new Nl2SqlResult(sql, (String) map.get("explanation"), data, "ai");
}
```

**用户**：万一 AI 生成错了或者超时呢？

**助手**：`callDeepSeek` 返回 null 时会走 `fallbackNl2Sql`，用关键词匹配做一些预定义的简单查询，不至于完全不可用。

### 对话 3：速率限制 + 多角色 AI 接口

**用户**：每个用户每天只能调用 10 次 AI，三个角色各一个接口，怎么办？

**助手**：写一个 `RateLimiter`，用 `ConcurrentHashMap` 按 userId 计数，`@Scheduled` 每天零时清零。

```java
@Component
public class RateLimiter {
    public static final int DAILY_LIMIT = 10;
    private final Map<Long, Integer> counter = new ConcurrentHashMap<>();

    public boolean tryAcquire(Long userId) {
        int count = counter.getOrDefault(userId, 0);
        if (count >= DAILY_LIMIT) return false;
        counter.put(userId, count + 1);
        return true;
    }

    @Scheduled(cron = "0 0 0 * * ?")
    public void reset() { counter.clear(); }
}
```

**用户**：Controller 层怎么控制角色权限？

**助手**：三个端点各检查 role 属性，再加限流判断。

```java
// 学生：故障分析
@PostMapping("/api/ai/student/analyze")
public Result<?> studentAnalyze(@Valid @RequestBody AiRequest req,
                                HttpServletRequest request) {
    if (!"STUDENT".equals(request.getAttribute("role")))
        return Result.error(403, "仅学生可用");
    if (!rateLimiter.tryAcquire(userId))
        return Result.error(429, "今日次数已用完");
    return Result.success(aiService.analyzeFault(req.getQuery()));
}

// 维修人员：方案推荐
@PostMapping("/api/ai/worker/suggest")
// 管理员：NL2SQL 查询
@PostMapping("/api/ai/admin/query")
// 查询剩余次数
@GetMapping("/api/ai/quota")
```

**用户**：三个角色的 AI 功能有什么区别？

**助手**：
- **学生** → 故障分析：拍个照描述问题，AI 判断可能原因
- **维修人员** → 方案推荐：输入故障现象，AI 给出维修建议
- **管理员** → NL2SQL：自然语言查数据库，统计型查询

## 五、前端 API 调用代码生成

### 对话 1：axios 封装 + JWT 令牌自动携带

**用户**：axios 怎么封装？要自动带 token，401 跳登录页。

**助手**：创建 `request.js`，用拦截器处理。

```js
// utils/request.js
import axios from 'axios'

const service = axios.create({
  baseURL: '/api',
  timeout: 15000
})

// 请求拦截器：自动带 token
service.interceptors.request.use(config => {
  const token = localStorage.getItem('token')
  if (token) {
    config.headers['Authorization'] = `Bearer ${token}`
  }
  return config
})

// 响应拦截器：401 跳登录
service.interceptors.response.use(
  response => response.data,
  error => {
    if (error.response?.status === 401) {
      localStorage.removeItem('token')
      window.location.href = '/login'
    }
    return Promise.reject(error)
  }
)

export default service
```

**用户**：`response.data` 直接返回，那调用方拿到的就是后端 Result 对象？

**助手**：对，所以组件里 `res.code`、`res.data` 直接用，不用再 `.data.data`。

### 对话 3：Vue 组件中调用 API — 评价功能

**用户**：评价功能前端怎么写？已完成未评价的显示星级打分，已评价的显示结果。

**助手**：用 `v-if`/`v-else` 区分两种状态，`el-rate` 组件做星级评分。

```vue
<template>
  <div v-if="isCompleted(item) && !item.rating" class="rating-section">
    <el-rate v-model="ratingForms[item.id]" :max="5" />
    <el-input v-model="reviewForms[item.id]"
              placeholder="说说你的评价..." type="textarea" />
    <el-button @click="submitRating(item.id)" :loading="ratingLoading">
      提交评价
    </el-button>
  </div>

  <div v-else-if="item.rating" class="rating-section">
    <el-rate v-model="item.rating" disabled :max="5" />
    <p>{{ item.review }}</p>
  </div>
</template>

<script setup>
import { reactive, ref } from 'vue'
import { rateRepair, getMyRepairs } from '@/api/repair'
import { ElMessage } from 'element-plus'

const ratingForms = reactive({})
const reviewForms = reactive({})
const ratingLoading = ref(false)

function submitRating(repairId) {
  ratingLoading.value = true
  rateRepair({
    repairId,
    rating: ratingForms[repairId],
    review: reviewForms[repairId]
  }).then(res => {
    ElMessage.success('评价成功')
    fetchRepairs()
  }).finally(() => {
    ratingLoading.value = false
  })
}

function isCompleted(item) {
  return item.status === '已完成'
}
</script>
```
